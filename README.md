# Task-Relevant Drift (TR-Drift)

**A Causal Account of Catastrophic Forgetting in LoRA Continual Fine-Tuning**

Haithem Barkaoui — Independent Researcher, France

This repository contains the full, reproducible code and the paper draft for our
study of catastrophic forgetting in LoRA continual fine-tuning of GPT-2.

---

## TL;DR

Catastrophic forgetting is often blamed on the **static geometric overlap**
between task subspaces. We show that this overlap is a **weak** predictor of
forgetting. We instead introduce **Task-Relevant Drift (TR-drift)** — the share
of backbone parameter motion that lands *inside the previous task's functional
subspace* — and show, with a controlled intervention, that it is a **direct
cause** of forgetting.

The story has four parts, all backed by reproducible experiments:

1. **Negative result.** Static subspace overlap only weakly predicts forgetting
   (r = 0.39, N = 162) and is largely redundant with total parameter motion.
2. **Correlation.** TR-drift predicts forgetting better (r = 0.63) and adds
   incremental validity after controlling for total drift
   (partial r = 0.52, p = 1.5e-12; ΔR² = +0.21).
3. **Decomposition.** Splitting total drift into task-relevant and orthogonal
   parts localizes the predictive signal to the task-relevant component
   (joint regression: TR coef = +127.1, p = 1.4e-12; orthogonal coef = −5.6).
4. **Causation.** At *matched* total step norm, steering the backbone *into* the
   previous task's subspace nearly doubles forgetting versus steering it
   orthogonally (0.186 vs 0.106; Cohen's d = 0.94, p = 1e-4, N = 30).

TR-drift also tracks forgetting **across** EWC, OGD, and GPM (r = 0.53,
p = 1.3e-5), indicating it is a general descriptor rather than an artifact of
plain fine-tuning.

> **Scope.** All results are on **GPT-2 small** and binary text-classification
> task pairs. Scaling the causal test to larger models is the main next step.

## Method in one paragraph

We fine-tune GPT-2 small with LoRA (rank 16) on the attention projections, with
the backbone trainable. After training Task A we estimate its **functional
subspace** P_A from the top principal components of Task-A backbone gradients.
After then training Task B, we measure **total drift** (norm of the backbone
change) and **TR-drift** (the part of that change lying inside P_A). The causal
test re-trains Task B while projecting the backbone update either *into* P_A or
*orthogonal* to it, rescaled to identical step norm, so the two arms differ only
in direction relative to P_A.
